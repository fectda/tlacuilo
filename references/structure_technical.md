# Estructura de Referencia: Atoms & Bits (Técnico)

Esta estructura sigue el formato "Spec Sheet" (Hoja de Especificaciones Técnicas), adaptando la metodología STAR (Situation, Task, Action, Result) para documentación de ingeniería.

## 1. 🎯 El Desafío (Situation/Task)
-   **Qué**: Definición del problema técnico o de negocio.
-   **Por qué**: El "dolor" o la ineficiencia que detonó el proyecto.
-   **Estilo**: Directo, sin rodeos. "El sistema era lento".

## 2. 💡 La Solución (Action - Engineering)
-   **Qué hiciste**: La arquitectura o el diseño.
-   **Cómo**: Decisiones técnicas clave. (Ej. "Usamos RabbitMQ porque...").
-   **Retos**: Obstáculos encontrados y cómo los superaste (Iteraciones).

## 3. 🛠️ Arquitectura / Especificaciones (Action - Tech)
-   **Lista de tecnologías (Stack)**.
-   **Diagramas** (si hay).
-   **Detalles de implementación** (Hardware, Software).

## 4. 🧪 Resultados (Result)
-   **Métricas**: Datos duros. "90% de reducción", "3x más rápido".
-   **Impacto**: Qué cambió en la realidad después de este proyecto.
-   **Veredicto**: ¿Funcionó? ¿Falló? (Honestidad brutal).
